redux

为什么要用redux? 当项目变得越来越大,每个组件都有自己的state的话,数据传递,一个组件数据变化后,可能引起另一个组件的变化.代码会很复杂,不方便理解和维护. 什么是redux? redux的核心概念: redux工作原理如下: 1. 首先整个项目有一个 store ,存放所有数据, 2. 某个组件通过this.state = store.getState(); 使用store中的数据; 要想修改 store 中的内容, 必须通过 action 来描述自己想要进行的操作;然后通过 store.dispatch(action); 把任务分给 reducer 3. reducer 只是一个接收 state 和 action，并返回新的 state 的函数。根据不同的action,进行相应的数据操作 三大原则 1. 单一数据源 只有唯一一个store 2. State 是只读的 唯一改变 state 的方法就是触发 action，action 是一个用于描述已发生事件的普通对象。 3. 使用纯函数来执行修改 需要编写 reducers。
